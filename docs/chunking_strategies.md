# 🧩 DeepVault Technical Chunking Architecture

This document provides a low-level technical breakdown of the four chunking strategies implemented in DeepVault. Our engine doesn't just slice text; it intelligently negotiates between character limits, sentence boundaries, and semantic meaning to maximize retrieval precision.

---

## 1. Fixed Window Chunker 
**Path**: `app/infrastructure/chunkers/fixed.py`

The bedrock of our ingestion pipeline. It provides a deterministic, high-speed baseline for comparing complex strategies.

### 🛠️ Technical Implementation
- **Validation Logic**: The class enforces strict constraints (via `ValueError`) to ensure `chunk_overlap < chunk_size`. This prevents "Infinite Window Hallucinations" where the pointer never advances.
- **Short Document Handling**: If the total content is less than the `chunk_size`, the algorithm automatically bypasses the loop and returns a single, cohesive chunk to preserve context.
- **Buffer Arithmetic**: Employs a `start += chunk_size - chunk_overlap` pointer movement to maintain contextual continuity across adjacent vectors.

### ⚙️ Parameters (Normalized)
*   **Chunk Size**: `500` characters.
*   **Overlap**: `100` characters (**20%**).
*   **Isolated Collection**: `deepvault_fixed`

---

## 2. Sliding Window Chunker (The Sentinel)
**Path**: `app/infrastructure/chunkers/sliding.py`

An advanced boundaries-aware engine that actively "hunts" for natural linguistic terminations.

### 🛠️ Technical Implementation
- **Graceful Expansion**: Unlike the Fixed chunker, this engine looks ahead **100 characters** beyond the ideal window. 
- **Sentence Hunting**: It utilizes a Regex Sentinel (`r"[.!?]\s"`) to identify the nearest logical ending. 
    - *If a break is found*: It expands the `effective_end` to include the full sentence.
    - *If no break is found*: it terminates exactly at the window size to prevent overflow.
- **Stride Control**: Moves the window forward by a fixed `stride`, creating a "rolling" context window that is highly resilient to mid-sentence retrieval cuts.

### ⚙️ Parameters (Tuned)
*   **Window Size**: `500` characters.
*   **Stride (The Step)**: `400` characters (**20% Overlap**).
*   **Isolated Collection**: `deepvault_sliding`

---

## 3. Semantic Chunker (The Topic-Aware)
**Path**: `app/infrastructure/chunkers/semantic.py`

Our most computationally intensive engine, shifting from character-counting to "meaning-counting."

### 🛠️ Technical Implementation
- **Regex Sentence Splitting**: Uses a lookbehind regex `r"(?<=[.!?])\s+"` to surgically split the document into discrete semantic units (sentences).
- **Similarity Matrix**: Computes a 1D similarity array between consecutive sentences using **Cosine Similarity** ($\cos(\theta) = \frac{A \cdot B}{\|A\| \|B\|}$).
- **Topic Shift Detection**: A new chunk is triggered only when the similarity drop falls below the `similarity_threshold`.
- **Tiny Chunk Suppression**: Implements a `min_chunk_size` (100 chars) filter. Any "micro-chunks" generated by short topic shifts are automatically merged into their preceding neighbor to ensure the resulting vector has enough semantic density to be useful.

### ⚙️ Parameters (Normalized)
*   **Similarity Threshold**: `0.75`.
*   **Min Chunk Size**: `100` characters.
*   **Isolated Collection**: `deepvault_semantic`

---

## 4. Structure Chunker (The Hierarchical)
**Path**: `app/infrastructure/chunkers/structure.py`

Explicitly modeled for Enterprise documentation, this engine treats Markdown syntax as a set of logical "hard-breaks."

### 🛠️ Technical Implementation
- **Heading Extraction**: Uses a multiline regex (`r"^(#{1,6})\s+(.+)"`) to identify Markdown headers.
- **Recursive Overflow Delegation**: This is the most complex logic in the suite.
    1. It extracts a logical section (e.g., `# Introduction` to `## Features`).
    2. It measures the section length against `max_section_size`.
    3. **If overflow occurs**: It doesn't just cut the text; it **recursively instantiates** a `FixedWindowChunker` specifically to handle the sub-splitting of that large section, ensuring sections remain bite-sized while preserving their headers.
- **Fallback Mode**: In documents without headers (e.g., standard text or scraped PDFs), it automatically degrades gracefully into a pure `FixedWindow` operation.

### ⚙️ Parameters (Normalized)
*   **Max Section Size**: `1500` characters.
*   **Fallback Size**: `500` characters.
*   **Isolated Collection**: `deepvault_structure`
